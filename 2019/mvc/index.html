MVC:拒绝面条式代码

Written by 周啟尧 with ♥ on December 14, 2019 in 技术

设计模式概述

设计模式是我们在进行程序设计的过程中，对普遍存在的问题（或反复出现的情况）提出的一种统一的解决方案。 比如适配器模式、Promise、重载、别名、getter/setter、观察者模式等。

代码模块化

如果代码中存在一些可以具体实现某些功能的代码块，而这些代码块的功能在其他的地方也可以适用。那么我们可以将这部分代码可以封装成一个函数、一个类，这就是代码的模块化；

根据操作对象的不同，比如将专门操作DOM对象的代码放在一类，将专门负责页面渲染更新的代码当在一类，这也是代码的模块化。

就是说，总是会有”万金油”的代码，是所有项目或块的通用代码，我们将这些具有相同”目的”的代码整合成一个模块，就可以使代码的整体结构更加清晰；模块之间虽然有联系，但每个模块只需关注自身的执行情况以及传进来的参数即可，不会牵一发而动全身，这也是一种解耦的思想。

MVC 设计模式

MVC 是设计模式的一种，是我们优化代码结构的一种方案。同时，他也是代码模块化的一种体现。

它不直接指导具体代码的实现，而是从全局的、宏观的角度，指导各个部分的代码应该怎么“摆”，从而使代码整体结构更加规律、直观。 当需求越是复杂，要实现的功能越复杂的时候，MVC 所带来的好处就越明显。

MVC 把代码的结构分成三个部分：Model-数据、View-视图、Control-控制器。

Model 数据

Model 负责代码中的所有与数据相关的部分，比如数据的更新、数据的读取、数据的存储等。 它不依赖于 View 和 Control，也就是说，它并不关心数据会被如何显示，或是被如何处理，它只需要管理好它的数据即可。 以下面的 Model 举例:

class Model{
    constructor(options){ // 接受数据
            const keys = ['data','update','create','delete','get']
        keys.forEach((key)=>{
            if(key in options){
                this[key]=options[key]
            }
        })
    }
    // 各种数据相关的方法
    create(){}
    delete(){}
    update(){}
    get(){}
}

View 视图

View 负责处理用户界面相关的代码，比如页面的重新渲染。 同样，View 并不关心它渲染的内容，它只要做好它的工作就可以了。 以下面的 代码 举例:

const view = {
    init(){} // 页面初始化
    template:`<section><p>hello</p></section>` // 页面要渲染的内容
    render(){} // 渲染函数
}

Controller 控制器

Controller负责统筹整个程序的运作，类似中央处理器。 比如对Model的数据进行处理，将处理后的数据通知给 View 更新；处理 View 传达的事件，更新数据后传达给 Model。 以下面的 Controller 举例:

class Controller{
    constructor(options) {
        Object.assign(this,options)
        this.el = $(this.el)
        this.autoBindEvents()  
        this.on('m:updated', () => {}) // 监听model中数据的变化，对页面进行重新渲染
    }
    autoBindEvents() {} // 自动绑定事件函数
}

通过 MVC 模式，可以将庞大的代码一个个分块，实现了一种动态的程序设计，使得后续对程序的修改和扩展简化。

什么是动态的程序设计呢？

比方说我们在一个只有加减乘除的页面计算器中增加一个求余的功能，我们只需要在 Controller 里面更改即可，因为我们根本不要需要知道数据是怎么得到的以及数据是怎么渲染到页面上，这些都会由 Model 和 View 处理。

表驱动编程

表驱动编程是可以对代码进行简化的一种编程模式。数据量少的时候我们可以采用逻辑语句，而数据量大的时候表驱动编程的优势就显现出来了。 遍历表中的元素，找到相对应的值提取出来，再插入到代码语句中使用即可。比如下面的自动绑定事件函数：

  bindEvents() {
    v.el.on('click', '#add1', () => {
        m.data.n += 1;
        localStorage.setItem("n", m.data.n.toString())
        v.render(m.data.n)
    });
    v.el.on('click', '#minus1', () => {
        m.data.n -= 1;
        localStorage.setItem("n", m.data.n.toString())
        v.render(m.data.n)
    });
    v.el.on('click', '#mul1', () => {
        m.data.n *= 2;
        localStorage.setItem("n", m.data.n.toString())
        v.render(m.data.n)
    });
    v.el.on('click', '#divide1', () => {
        m.data.n /= 2;
        localStorage.setItem("n", m.data.n.toString())
        v.render(m.data.n)
    });
}

上面的代码中有许多相同的部分，我们抽离出其中不同的部分做成表，只留下必要的“骨架”，然后再遍历表对骨架进行填充。

events : {
    'click #add1':'add',
    'click #minus1':'minus',
    'click #mul1':'mul',
    'click #divide1':'divide',
},

add(){
    m.update({n:m.data.n + 1})
},
minus(){
    m.update({n:m.data.n + 1})
},
mul(){
    m.update({n:m.data.n * 2})
},
divide(){
    m.update({n:m.data.n / 2})
},

autoBindEvents(){
    for(let key in c.events){
        const value = c[c.events[key]]
        const spaceIndex = key.indexOf(' ')
        const part1 = key.slice(0,spaceIndex)
        const part2 = key.slice(spaceIndex
        +1)
        c.el.on(part1,part2,value) // 逐一绑定事件
}

此外，还可以这么简化

class Model extends EventBus{
    constructor(options){
        super() // 调用EventBus#constructor
        this.data = data
        this.update = update
        this.create = create
        this.delete = delete
        this.get = get
}
//=================================================
class Model extends EventBus{
    constructor(options){
        super() // 调用EventBus#constructor
            const keys = ['data','update','create','delete','get']
        keys.forEach((key)=>{
            if(key in options){
                this[key]=options[key]
            }
        })
}

总而言之，当面对结构类似的代码，我们只写一遍，抽离其中的精髓，放到一个表中，然后再遍历这个表，将表中的关键词再插入到代码中。

EventBus

EventBus也是一种设计模式，主要用于对象/组件之间的通信，我们以MVC模式举例。

比如当我们点击button时，页面中的数字+1。这样的功能在MVC模式中要如何实现呢？

我们需要实现m模块与v模块之间的通信，有两种方法可以实现：

  1. 使用vue2的api: Object.defineProperty
  2. 使用eventBus。

下面简单阐述一下eventBus的使用。
我们可以通过jquery模拟一个EventBus。

import $ from 'jquery'

const eventBus = $(window)

这里创建的eventBus包含了on方法用于监听事件，trigger方法用于触发事件。

const m = new Model({
        data: {
            n: parseInt(localStorage.getItem("n") || 100)
        },
    update(data){
        Object.assign(m.data,data)
        eventBus.trigger('m:updated') // 触发更新事件
    }
})

 init(container) {

        view.el = $(container)
        view.render(m.data.n)
        view.autoBindEvents()
        eventBus.on('m:updated',()=>{
            view.render(m.data.n)
            localStorage.setItem("n", m.data.n.toString())
        }) // 当trigger事件触发,重新渲染页面
    },
    .....
    .....
}

当Model中的数据发生变化，通过eventBus.trigger()触发一个”已更新”事件，然后用eventBus.on()监听这个事件，当”已更新”事件触发，重新渲染页面这样就实现了模块间的通信。

eventBus 的 API

除了on方法、trigger()方法，eventBus还有click()、off()、pretend()等API。我们用on监听事件，用trigger触发事件。
