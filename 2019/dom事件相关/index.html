DOM事件相关

Written by 周啟尧 with ♥ on December 8, 2019 in JavaScript

什么是事件委托？ 事件委托的原理是事件冒泡。 js的事件流分为三个阶段：事件捕获、目标阶段、事件冒泡。 捕获阶段：事件开始由不太具体的节点接收，然后逐级向下传播到最具体的节点。 目标阶段：在目标节点上触发。 冒泡阶段：事件开始由最具体的元素接收，然后逐级向上传播到较为不具体的节点（或文档）

所谓事件委托就是把原本需要绑定在子元素的响应事件委托给它的父元素，让父元素承担监听事件的任务。 比如有这么个结构：div > ul > li > a，我们可以给div添加点击事件，那么当div中的ul、li、a做点击事件的时候，会冒泡到最外层的div，触发div上的点击事件。

举例说明：

<ul id = 'list'>
    <li>item 1</li>
    <li>item 2</li>
    <li>item 3</li>
    <li>item 4</li>
</ul>

实现一个功能：点击li，弹出’1234’ 如果不用事件委托，我们需要把点击后弹出’1234’这个事件委托给每一个li

window.onload = () => {
    let ul1 = document.querySelector('#list')
    let lis =  ul1.getElementsByTagName('li')
    for(let i=0; i<lis.length; i++){
        lis[i].onclick = () => {
            alert('1234')
        }
    }
}

如果用事件委托，我们只需把事件委托给ul即可

window.onload = () => {
    let ul1 = document.querySelector('#list')
    ul1.onclick = () => {
        alert('1234')
    }
}

当li被点击时，由于事件冒泡原理，事件会传递给li的父元素ul，ul上挂着的事件就会触发。

怎么阻止默认动作？ 用e.preventDefault()，兼容IE用e.returnValue = false

function stopDefault(e){
    let e = e || window.event;
    if(e && e.preventDefault){
        e.preventDefault();
    }else{
        e.returnValue = false;
    }
    return false;
}

怎么阻止事件冒泡？ 用e.stopPropagation(),兼容IE则需要使用e.cancelBubble = true

function stopPropagation(e){
    let e = e || window.event;
    if(e && e.stopPropagation){
        e.stopPropagation();
    }else{
        e.cancelBubble = true;
    }
}
