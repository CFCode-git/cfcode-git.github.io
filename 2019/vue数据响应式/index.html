Vue:数据响应式

Written by 周啟尧 with ♥ on December 20, 2019 in 技术

前言

先说两个基本的认识

响应式：若某个东西能对外界的变化(刺激)做出反应，那么他就是响应式的。
比如响应式页面，就是当我们改变窗口的大小时，网页内容会根据窗口的大小做出反应(调整)的页面。

数据响应式： Vue的data是响应式的。比如const vm = new Vue({data:{n:0}})，
如果修改vm.n(或data.n)，那么页面中的n就会作出响应，渲染为对应的n或者经过一些数据处理后的n。这就是响应式
Vue2 通过 Object.defineProperty 实现数据响应式。

接下来，理解下面两个ES6语法，有助于本文的阅读。

1. getter/setter

现有如下对象

let obj = {
    firstName:'张',
    lastName:'三',
    name(){
        return this.firstName + this.lastName
    }
}

console.log(obj.name())
// 张三

通过调用obj的name函数就可以得到这个对象的姓名。

  • getter/setter

    let obj = {
firstName:'张',
lastName:'三',
get name(){
    return this.firstName + this.lastName
},
set name(arr){
    this.firstName = arr[0]
    this.lastName = arr.substring(1)
}
}

console.log(obj.name)
obj.name = '李四'
console.log(obj.name)
// 张三
// 李四

使用get方法和set方法，name不再是一个函数，它是一个计算属性，它的值是以函数的形式定义的。
当我们调用name的时候，就可以通过get得到name这个属性计算的值。比如本例中计算的值就是姓名：”张三”
当我们给name赋值的时候，就可以通过set进行赋值，比如本例中的”李四”就是重新赋值的结果。

打印出obj对象，结果是这样：

# {firstName: "李", lastName: "四"}
# firstName: "李"
# lastName: "四"
# name: (...)
#  get name: ƒ name()
#  set name: ƒ name(arr)
#  __proto__: Object

可以看到name显示结果为(…)
结论是：我们确实可以对name进行读和写，但是并不存在一个叫name的属性，这个读和写是通过 get 和 set 实现的。
记住这个结论。

2. Object.defineProperty

接上面的例子，get和set是在对象声明的时候使用的，无法在一个对象声明结束后添加getter/setter；
Object.defineProperty 就是解决这个问题的方案。

在上面的代码中加入age属性，对age进行getter/setter。

let obj = {
    firstName:'张',
    lastName:'三',
    get name(){
        return this.firstName + this.lastName
    },
    set name(arr){
        this.firstName = arr[0]
        this.lastName = arr.substring(1)
    },
    age:18
}

Object.defineProperty(obj,'num',{
    get(){
        return obj.age
    },
    set(value){
        obj.age = value
    }
})
  1. obj 是需要添加getter/setter的对象；
  2. ‘num’是操作的属性名，但需要注意的是，它不存在于obj对象中，本例中获取的是age的值，这个getter/setter的属性名也不能与age重名，否则会造成死循环。

2. Vue是如何做到让data无法逃脱Vue的监控的？

根据上面的事实，Vue对数据进行了处理，myData对象传入Vue之后，经过处理就得到了n:(…)这么个东西。
Vue知道myData的值发生了改变，数据响应式，改变后的结果也被渲染到页面上。

接下来我们通过一个案例理解

我们假设有一个对象data，我们规定data.n不能小于0，
可以用getter/setter解决这个需求。

let data = {}

data._n = 0  // 用来存储 n 的值

Object.defineProperty(data,'n',{
    get(){
        return this._n
    },
    set(value){
        if(value < 0) return
        this._n = value
    }
})

data是一个空对象，里面没有n，我们用data._n保存n的值。
当设置的值小于0，就会被set忽略。


问题是，我们虽然无法给data.n设置一个小于0的值，但我们可以直接给存储n的data._n设置负值，那么getter/setter机制就被绕过了。
解决办法：使用代理。

代理是一种设计模式，先上代码

let data = proxy({data:{n:0}})

function proxy({data}){
    const obj = {}
    Object.defineProperty(obj,'n',{
    get(){
        return data.n
    },
    set(value){
        if(value < 0) return
        data._n = value
    }
    })
    return obj
}

上面代码中，我们将数据作为参数传递给proxy函数，由proxy函数创建一个空对象obj，这个obj就是代理，返回这个代理给data。
这样就可以不暴露数据，同时通过代理实现了data的getter/setter。


问题是，这个方法并不完美，如果传入的数据有名字，我们一样可以从外部更改数据，绕过代理。

解决办法：监听data

let myData = {n:0}
let data5 = proxy({ data:myData })

function proxy({data}){
//   let value = data.n
//   Object.defineProperty(data, 'n', {
//     get(){
//       return value
//     },
//     set(newValue){
//       if(newValue<0)return
//       value = newValue
//     }
//   })
  const obj = {}
  Object.defineProperty(obj, 'n', {
    get(){
      return data.n
    },
    set(value){
      if(value<0)return
      data.n = value
    }
  })
  return obj
}

上面代码中，给数据取了名字为myData，然后就可以通过myData设置负值，绕过代理中的setter。


注释内容是给data设置的监听。

具体思路就是：
当myData传给proxy函数时，拿到myData中的数据，并给myData添加getter/setter。
当要通过外部篡改myData中的n，实际上此时的myData已经被proxy篡改，传入的值会经过myData中的set检验，如果不符合要求则无法修改。
也就是说data无法逃脱proxy的监控了，data做的任何修改都会被proxy捕获到。


上面例子中的proxy函数就相当于是Vue做的处理。
当我们vm = new Vue({data:myData})时，Vue会做两件事情。
1. 让vm成为myData的代理；
2. 对myData的所有属性实施监控，以便属性改变时，可以及时render

Vue的data有bug？

Vue可以对data中的对象进行监听和代理，但是Vue无法进行事先的监听和代理。

就是说，没有出现在data中的对象，Vue是无法操作的。

如果我们在初始化后发现要添加数据属性怎么办？

解决方案有两种：
1. 事先在data中加上新的属性。 2. 使用Vue的两个API Vue.set(object,key,value) 和 this.$set(object,key,value)(效果一样，避免重名用后者,this指的就是Vue实例)

用法：

new Vue({
    el:'xxx'
    data:{
        obj:{
            a:0,
        }
    },
    template:`......`
    methods:{
        setB(){
            Vue.set(this.obj,'b',1)
            //this.$set(this.obj,'b',1)
        }
    }
})
