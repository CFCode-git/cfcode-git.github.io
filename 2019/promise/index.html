Promise

Promise 是目前前端异步编程的一个解决方案。

它解决了回调地狱的问题，规范了回调的名字和顺序，同时也方便了捕获错误。

一个 Promise 有三种状态：

pending：表示初始状态，既没有成功，也没有失败

fulfilled 表示操作成功完成

rejected 表示操作失败。

创建一个 Promise

// 直接用Promise构造函数创建一个Promise对象
let promiseFn = new Promise((resolve,reject)=>{})

// 也可以在回调的异步函数的第一行加上下面这句代码
function promiseFn(){
  return new Promise((resolve,reject)=>{...})
}

Promise.then()

Promise.then()接受两个回调函数，第一个为成功回调，第二个为失败回调。

var p1 = new Promise((resolve, reject) => {
  resolve("成功！");
  // or
  // reject(new Error("出错了！"));
});

p1.then(
  (value) => {
    console.log(value); // 成功！
  },
  (reason) => {
    console.error(reason); // 出错了！
  }
);

Promise.all()

接受多个 Promise 组成的可迭代对象，比如数组。返回结果是一个新的 Promise 对象。

当所有 Promise 都成功时，将所有 Promise 的返回值组成的数组作为成功回调的返回值。

只要有一个失败，将第一个失败的 Promise 的错误信息作为它的错误信息，执行失败回调。

/* Promise.all 成功*/
var p1 = Promise.resolve(3);
var p2 = 1337;
var p3 = new Promise((resolve, reject) => {
  setTimeout(resolve, 100, 'foo');
});

Promise.all([p1, p2, p3]).then(values => {
  console.log(values); // [3, 1337, "foo"]
});

/* Promise.all 失败*/
var p1 = new Promise((resolve, reject) => {
  setTimeout(resolve, 1000, "one");
});
var p2 = new Promise((resolve, reject) => {
  setTimeout(resolve, 2000, "two");
});
var p3 = new Promise((resolve, reject) => {
  setTimeout(resolve, 3000, "three");
});
var p4 = new Promise((resolve, reject) => {
  setTimeout(resolve, 4000, "four");
});
var p5 = new Promise((resolve, reject) => {
  reject("reject");
});

Promise.all([p1, p2, p3, p4, p5]).then(
  (values) => {
    console.log(values);
  },
  (reason) => {
    console.log(reason);
  }
);

// 返回结果：From console:
//"reject"

Promise.race()

接受多个 Promise 组成的可迭代对象，比如数组。返回结果是一个新的 Promise 对象。

Promise.race()的成功/失败取决于第一个执行完成的 Promise。

用下面的例子理解： 传入两个 Promise 对象组成的数组，输出第一个执行完毕的 Promise 的结果。

// 声明变量promise1，用构造函数Promise构造一个Promise对象并赋值给promise1，该对象在500毫秒后调用resolve函数。
let promise1 = new Promise(function (resolve, reject) {
  setTimeout(resolve, 500, "one");
});

// 声明变量promise2，用构造函数Promise构造一个Promise对象并赋值给promise2，该对象在100毫秒后调用resolve函数。
let promise2 = new Promise(function (resolve, reject) {
  setTimeout(resolve, 100, "two");
});

// 传入两个对象异步执行，他们都成功，resolve会调用then后面的成功函数。而promise2先出结果，因此输出promise2的结果。
Promise.race([promise1, promise2]).then(function (value) {
  console.log(value);
  // Both resolve, but promise2 is faster
});
// expected output: "two"

Promise 的缺点

  1. 无法取消 Promise。一旦新建他就会立即执行，无法中途取消。

  2. 如果不设置回调函数，那么 Promise 内部抛出的错误，不会反映到外部。

  3. 处于 pending 状态时，无法得知目前发展到哪个阶段。（刚刚开始还是即将完成）
