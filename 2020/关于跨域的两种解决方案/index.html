关于跨域的两种解决方案

Written by 周啟尧 with ♥ on April 14, 2020 in JavaScript

浏览器为了保护用户隐私，设计了同源策略。即：不同源的页面之间不能互相访问数据。

什么是同源？即相同的协议、域名和端口号。

有时候我们需要获取别的网站的数据，这时候就需要用到跨域。这里主要说两种解决方案。

一、CORS 跨域资源共享

如果浏览器支持 CORS 就很简单。只需要后台在响应头里面设置一下即可：

response.setHeader("Access-Control-Allow-Orogin", "http:example.com:9999");

不过 CORS 并不支持 IE6 7 8 9。。

如果需要兼容 IE，我们可以用 JSONP。

二、JSONP

虽说数据不能跨域，但 JS 文件是没有同源的限制的。我们大可把数据放到 JS 文件中。

具体怎么做呢？

首先，提供数据的一方需要做的是：

  1. 将数据写到 JS 文件里。比如【data.js】

    // 可以是赋值，也可以是函数
 window['{{xxx}}']({{data}})

  2. 在收到这个 data.js 的请求的时候，获取 data.json 的数据，替换{{data}}；

  3. 用 query 获取请求的参数【一般放在 callback 里面，是回调函数名。】用来替换{{xxx}}。

  4. 然后将替换后的结果返回出去即可。

else if(path="/data.js"){
	response.statusCode = 200;
	response.setHeader('Content-Type','text/javascript;charset=utf-8')
	const string = fs.readFileSync('./public/friends.js').toString()
	const data = fs.readFileSync('./public/friends.json').toString() /* 真正的数据 */
	const string2 = string.replace('{{ data }}', data).replace('{{xxx}}',query.callback)  /* 替换 */
	response.write(string2)
	response.end()
}

获取数据的一方:

  1. 用 script 标签引用 data.js

  2. 定义 callback 给 data.js 回调。

  3. 这里做了两个优化：

  • 为了避免全局变量冲突，我们可以使用随机数作为 callback。

  • 拿到数据就删除 script 标签，避免页面中的 script 标签越来越多。

const = random = Math.random()
console.log(random)
window[random]= ()=>{
    console.log(data)
}
const script = document.createElement('script')
script.src =  `http://qq.com:8888/friends.js?callback=${random}`  /* 用查询参数传随机数 */
script.onload=()=>{
    script.remove()      /* 执行完删除 script */
}
document.body.appendChild(script)

通过 JSONP 就可以在兼容 IE 的情况下实现跨域啦。

三、总结

当我们需要跨域，优先使用 CORS 是最便捷的。通过设置请求头Access-Control-Allow-Origin

而当浏览器不支持 CORS 的时候，可以采用 JSONP 跨域。

我们请求一个 JS 文件，这个 JS 文件会执行一个回调，回调里面就有我们的数据。

回调的名字可以是一个随机生成的随机数，我们把函数名以 callback 的参数传给目标源的后台，后台将 JS 文件中对应的部分替换成我们的 callback 参数后，再把函数返回给我们执行。

四、JSONP 的优缺点

JSONP 的可以兼容 IE 跨域，他的缺点是：

  1. 由于是 script 标签，因此无法读取精确的状态，比如状态码、响应头等。他只有成功和失败。

  2. 由于它是 script 标签，因此他只能发 get 请求，不支持 post。
